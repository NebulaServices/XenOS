# Rules

## Dev-Only

-   Don't make any major changes without first notifying the team

# Variables

-   Properly name your variables (Use e only for events, use underscore only for unused variables, ...)
-   Use the variable types properly (No var when you could use const)
-   Don't declare variables that aren't used
-   Put variable assignments before element modifiers

# Logging

-   Don't forget to delete your loggers that you're done with

# Documentation

-   Document complex code (Such as using many monads)
-   Before you commit incomplete code, document it
-   Write proper annotations for apis

# General

-   Don't use setTimeout if you don't have to (If you're not timing something)
-   No evals
-   No timeouts unless absolutely necessary (Learn how to use async code)
-   No "== true" in booleans
-   Don't forget to remove your debug logs
-   Test and format your code before you commit
